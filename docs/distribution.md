# Distribution

This repository is **offline-first**.

Every Photoshop plugin in this repository must ship with both a verified CCX and an offline package. The offline zip must contain the same CCX plus complete plugin files and Chinese instructions.

The offline package is a project requirement because users may not be able to reliably stay signed in to Creative Cloud, verify an Adobe account, complete online authorization, or open Adobe UXP Developer Tool without a login prompt. Offline packages must not depend on remote runtime assets.

## Release Priority

1. Verified `.ccx`.
2. Offline `.zip` containing that CCX, complete plugin files, Chinese instructions, install scripts, uninstall scripts, and install checks.
3. Adobe UXP Developer Tool loading only as a developer fallback, not as the expected user path.

See `docs/install-findings.md` before adding or changing installation paths.

## CCX Rule

Do not create CCX files by renaming or manually zipping a plugin folder.

All CCX release filenames use this project-wide convention:

```text
Dino-<plugin-name>_<host>.ccx
```

Example:

```text
Dino-layer-quick-jump_PS.ccx
```

Creative Cloud Desktop validates CCX package structure. Manually compressed `.ccx` files can fail with install error code `-4`.

The repository packager reproduces Adobe UXP Developer Tool's package transformation and validates the result with `scripts/verify-ccx.mjs`. A CCX must be installation-tested before release.

## UXP Offline Status

Directly copying UXP folders into user-level or system-level Adobe UXP extension folders did not make `layer-quick-jump` appear in Photoshop 2025. Keep those scripts only as diagnostics unless later testing proves a working variant.

CEP user-level copy also failed to show `layer-quick-jump-cep` in Photoshop 2025. If CEP remains the direction, test system-level CEP installation before treating it as a working offline path.

## Chinese Install Notes

Chinese installation instructions are mandatory for every plugin release.

- Each plugin must include `INSTALL.md` written in Chinese.
- Every offline `.zip` must include a root-level `INSTALL_OFFLINE.md` written in Chinese.
- Every offline `.zip` must also include a root-level Chinese plain-text `安装说明.txt`.
- Every offline `.zip` must also include `INSTALL_OFFLINE_CN.txt` for systems that do not display Chinese filenames reliably.
- Every offline `.zip` must include `OFFLINE_FIRST.md` and `离线优先说明.txt`.
- Every offline `.zip` must include the same version's verified `.ccx`.
- Every offline `.zip` must include executable `INSTALL_CCX.command`.
- Every offline `.zip` must include `INSTALL_LOCAL.command` for the user-level local install experiment.
- Every offline `.zip` must include `INSTALL_SYSTEM.command` for the system-level local install experiment.
- Every offline `.zip` must include matching uninstall scripts for local experiments.
- Every offline `.zip` must include `CHECK_INSTALL.command` for support/debugging.
- The instructions must cover the preferred CCX install path, user-level local script install, system-level local script install, and the UXP Developer Tool fallback.
- Do not publish an offline package that only contains English install instructions.

## Build Artifacts

Package the offline release and CCX instructions from the repository root:

```bash
npm run package:plugin -- <plugin-name>
```

For the first plugin:

```bash
npm run package:layer-quick-jump
```

Artifacts are written to:

```text
dist/<plugin-name>/<version>/
```

The script creates:

- `Dino-<plugin-name>_PS.ccx`
- `<plugin-name>-v<version>-offline.zip`
- `生成CCX说明.txt`

## Release Checklist

- `manifest.json` has stable `id`, `name`, `version`, `main`, and `host` fields.
- The plugin has a plugin-specific `README.md`.
- The plugin has a plugin-specific Chinese `INSTALL.md`.
- The offline `.zip` contains Chinese `INSTALL_OFFLINE.md`.
- The offline `.zip` contains Chinese `安装说明.txt`.
- The offline `.zip` contains Chinese `INSTALL_OFFLINE_CN.txt`.
- The offline `.zip` contains `OFFLINE_FIRST.md` and `离线优先说明.txt`.
- The offline `.zip` contains executable `INSTALL_LOCAL.command`.
- The offline `.zip` contains executable `INSTALL_SYSTEM.command`.
- The offline `.zip` contains executable uninstall scripts.
- The offline `.zip` contains executable `CHECK_INSTALL.command`.
- The local script install path has been tested on the target Photoshop version, or clearly marked as experimental.
- If a CCX is shipped, it was generated by Adobe UXP Developer Tool and double-click tested on a local machine with Photoshop installed.
- If UXP Developer Tool loading is documented, it is clearly labeled as a developer fallback.
- No network-only runtime assets are required.
- Generated release artifacts stay out of Git unless a release process explicitly asks for them.
