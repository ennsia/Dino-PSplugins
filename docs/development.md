# Development Notes

## Local Workflow

1. Edit a plugin inside `plugins/<plugin-name>/`.
2. Run `npm run lint`.
3. Build the offline package with `npm run package:plugin -- <plugin-name>`.
4. Check `docs/install-findings.md` for known Adobe install path failures.
5. Test the offline install scripts first.
6. Use Adobe UXP Developer Tool only as a developer fallback when login state allows it.
7. Test inside Photoshop with a disposable document.

## Publishing Checklist

- The plugin folder has its own README and usage notes.
- The plugin folder has its own Chinese INSTALL.md.
- `manifest.json` has a stable `id`, `name`, and `version`.
- Build an offline `.zip` artifact when preparing any release.
- The offline `.zip` contains Chinese Markdown and txt instructions.
- The offline `.zip` contains local/system install scripts, uninstall scripts, and install checks.
- If a `.ccx` artifact is shipped, it is optional and generated by Adobe UXP Developer Tool.
- No private files, API keys, PSD/PSB files, or local-only paths are committed.
- The plugin has been tested in Photoshop through the offline path, or the offline path is clearly marked as blocked by Adobe/Photoshop behavior.
- For simple panels, CEP is still experimental: user-level CEP copy did not show in Photoshop 2025, so system-level CEP must be tested before calling it viable.
