Layer Quick Jump / 图层快速跳转
Design and Usage Notes / 设计与使用说明

Status / 状态

Version: 0.1.0 beta
This plugin has not been fully tested in real production files. It should be treated as an experimental personal workflow tool.

版本：0.1.0 beta
这个插件还没有在真实项目文件中完整跑测，应当视为实验性的个人流程工具。


1. What This Plugin Does / 插件用途

Layer Quick Jump is a small Photoshop UXP panel for saving three frequently used layer or layer-group positions in the current document.

图层快速跳转是一个小型 Photoshop UXP 面板，用于在当前文档中保存三个常用图层或图层组位置。

The basic workflow is:

1. Open a Photoshop document.
2. Select a layer or layer group.
3. Click one of the Save buttons.
4. Continue working elsewhere in the layer stack.
5. Click the matching Jump button to select that saved layer again.

基本使用流程：

1. 打开 Photoshop 文档。
2. 选中一个图层或图层组。
3. 点击某个“保存”按钮。
4. 继续在其他图层位置工作。
5. 点击对应的“跳转”按钮，重新选中之前保存的图层。


2. Why It Exists / 构思来源

Large ACG and game-art Photoshop files often contain many folders, draft layers, cleanup layers, color layers, masks, notes, and temporary merge layers.

大型 ACG / 游戏美术 Photoshop 文件中，常常会包含大量文件夹、草稿图层、清理图层、上色图层、蒙版、备注层和临时合并层。

In this kind of file, a lot of attention is lost not because the painting decision is difficult, but because the artist has to repeatedly scroll, expand folders, collapse folders, find the same layer again, and return to a working anchor point.

在这种文件里，很多注意力不是消耗在绘制判断本身，而是消耗在反复滚动图层、展开文件夹、折叠文件夹、重新找到同一个图层，以及返回某个工作锚点上。

The first idea of this plugin is intentionally simple: provide a few temporary memory slots for layer positions, similar to bookmarks inside the Photoshop layer panel.

这个插件的第一版想法非常简单：给图层位置提供几个临时记忆槽，类似 Photoshop 图层面板里的书签。

It does not try to manage the whole document. It only tries to reduce one small repeated action: "Where was that layer again?"

它不试图管理整个文档，只解决一个很小但反复出现的问题：“刚才那个图层在哪里？”


3. Current Structure / 当前结构

Plugin folder:

plugins/layer-quick-jump/

Important files:

- manifest.json
  Photoshop UXP plugin manifest. Defines the plugin id, name, version, target Photoshop host, and panel entrypoint.

- index.html
  Panel markup. Defines the title, three record slots, Save buttons, Jump buttons, and status message area.

- src/styles.css
  Panel styling. Keeps the UI compact enough for a docked Photoshop panel.

- src/main.js
  Main plugin logic. Handles record storage, active-layer reading, layer-tree search, and layer selection through Photoshop UXP APIs.

- docs/design-and-usage.zh-en.txt
  This note. Describes how the plugin is intended to be used and modified.


插件目录：

plugins/layer-quick-jump/

关键文件：

- manifest.json
  Photoshop UXP 插件清单。定义插件 id、名称、版本、目标 Photoshop 宿主和面板入口。

- index.html
  面板结构。定义标题、三个记录槽、保存按钮、跳转按钮和状态提示区域。

- src/styles.css
  面板样式。让 UI 在 Photoshop 停靠面板中保持紧凑。

- src/main.js
  插件主逻辑。负责记录存储、读取当前选中图层、遍历图层树，以及通过 Photoshop UXP API 重新选中图层。

  本说明。用于描述插件的使用方式和改造思路。


4. Implementation Notes / 实现说明

Record storage:

The plugin stores records in localStorage with the key:

layerQuickJump.records.v1

记录存储：

插件使用 localStorage 保存记录，key 为：

layerQuickJump.records.v1

Each record stores:

- layer id
- layer name
- layer path
- document id
- document name
- saved timestamp

每条记录保存：

- 图层 id
- 图层名称
- 图层路径
- 文档 id
- 文档名称
- 保存时间

Layer lookup:

The plugin recursively walks the document layer tree and compares layer ids. This allows it to find layers inside nested groups.

图层查找：

插件会递归遍历当前文档的图层树，并通过图层 id 对比目标。这使它可以找到嵌套组里的图层或图层组。

Jump action:

When the user clicks Jump, the plugin calls Photoshop batchPlay inside executeAsModal and selects the saved layer by id.

跳转动作：

用户点击“跳转”时，插件会在 executeAsModal 中调用 Photoshop batchPlay，通过保存的图层 id 重新选中该图层。

Document guard:

The plugin stores the document id and document name. If the current document is different from the saved document, it warns the user instead of jumping blindly.

文档保护：

插件会保存文档 id 和文档名称。如果当前文档和记录所属文档不一致，插件会提示用户，而不是直接跳转。


5. Current Limitations / 当前限制

- Only three record slots are available.
- Records are local to the plugin panel storage, not embedded into the PSD file.
- If a layer is deleted, duplicated into another document, or heavily reorganized, the saved id may no longer point to a valid target.
- Multi-layer selection is not the main target of this beta version.
- The plugin has not been tested across many Photoshop versions or large production-like documents.
- The current UI is intentionally minimal and may change.

- 当前只有三个记录槽。
- 记录保存在插件面板本地存储中，不会写入 PSD 文件。
- 如果图层被删除、复制到其他文档，或经过大规模重组，保存的 id 可能不再指向有效目标。
- 多图层选择不是这个 beta 版本的主要目标。
- 插件尚未在大量 Photoshop 版本或大型仿生产文件中测试。
- 当前 UI 刻意保持最小化，后续可能随时调整。


6. Ideas For Modification / 后续改造方向

Possible next steps:

- Rename the public module direction to Layer Locator while keeping this beta plugin as layer-quick-jump.
- Add more slots or configurable slot counts.
- Add custom labels for each saved record.
- Add a clear button for each slot.
- Add import/export for saved records.
- Add document-aware record groups.
- Add search-based fallback if a layer id becomes invalid but the layer path still exists.
- Add visual highlighting after jump.
- Add keyboard shortcuts if the UXP host workflow allows it.

可能的后续方向：

- 对外方向命名为 Layer Locator，同时保留当前 beta 插件名 layer-quick-jump。
- 增加更多记录槽，或允许自定义记录槽数量。
- 为每个记录点增加自定义标签。
- 为每个记录点增加清除按钮。
- 增加记录导入/导出。
- 按文档分组保存记录。
- 当图层 id 失效但路径仍然存在时，增加基于路径或名称的回退查找。
- 跳转后增加视觉提示。
- 如果 UXP 宿主流程允许，增加快捷键支持。
